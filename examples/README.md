# Examples

These files exist solely to give the CLI and the Tauri shell something synthetic to chew on. They are **not** real wordlists and the digests are **not** real-world credentials — do not use them for any real audit. Plaintexts are random ASCII strings; collisions with actual passwords are accidental.

## Files

### Scenario 0 — tiny (Phase 1 prototype)

- `tiny_dict.txt` — 10 common-knowledge passwords.
- `sample_hashes.txt` — 10 MD5 digests, one per line, in the same order as `tiny_dict.txt`.

The original smoke-test corpus from Phase 1. Use it for the canonical `gpuhash attack --algo md5 --hashes examples/sample_hashes.txt --wordlist examples/tiny_dict.txt --i-own-these-hashes` command.

### Scenario 1 — medium (algorithm comparison)

- `medium_dict.txt` — 1 000 unique random lowercase 4–8 char candidates.
- `medium_md5.txt` — 50 planted MD5 digests.
- `medium_sha1.txt` — same 50 plaintexts as SHA-1.
- `medium_sha256.txt` — same 50 plaintexts as SHA-256.

The three hash files use the **same plaintext set**, so you can directly compare CPU↔GPU throughput across algorithms without changing any other variable.

```powershell
# Audit MD5
gpuhash attack --algo md5    --hashes examples/medium_md5.txt    --wordlist examples/medium_dict.txt --i-own-these-hashes
# SHA-1
gpuhash attack --algo sha1   --hashes examples/medium_sha1.txt   --wordlist examples/medium_dict.txt --i-own-these-hashes
# SHA-256
gpuhash attack --algo sha256 --hashes examples/medium_sha256.txt --wordlist examples/medium_dict.txt --i-own-these-hashes
```

Each finds 50/50 matches.

### Scenario 2 — big (GPU dictionary)

- `big_dict.txt` — 50 000 unique random lowercase 4–8 char candidates.
- `big_md5.txt` — 200 planted MD5 digests drawn from the dictionary.

Large enough to keep the GPU saturated past pipeline startup tax.

```powershell
gpuhash attack --algo md5 --hashes examples/big_md5.txt --wordlist examples/big_dict.txt --gpu --i-own-these-hashes
```

Finds 200/200 matches.

### Scenario 3 — bruteforce (mask `?l?l?l?l`)

- `brute_4lower_md5.txt` — 50 MD5 digests of random 4-char lowercase plaintexts.

No wordlist needed — drive it with the 4-lowercase mask (keyspace 26⁴ = 456 976):

```powershell
gpuhash attack --algo md5 --hashes examples/brute_4lower_md5.txt --mask "?l?l?l?l" --gpu --i-own-these-hashes
```

Finds 50/50 matches in well under a second on a typical Intel iGPU — a good "this is what the GPU pipeline can do" demo.

### Scenario 4 — longer bruteforce (mask `?l?l?l?l?l`)

- `brute_5lower_md5.txt` — 50 MD5 digests of random 5-char lowercase plaintexts.

Same idea as Scenario 3 but with a bigger keyspace (26⁵ ≈ 11.9 M candidates), so the GPU pipeline runs long enough for the Tauri shell's live H/s chart to settle on a steady-state value:

```powershell
gpuhash attack --algo md5 --hashes examples/brute_5lower_md5.txt --mask "?l?l?l?l?l" --gpu --i-own-these-hashes
```

Finds 50/50 matches; measured ~123 MH/s sustained on an Intel UHD iGPU in ~0.1 s.

## Regenerating

### tiny scenario

If you change `tiny_dict.txt`, regenerate `sample_hashes.txt`:

```powershell
Get-Content examples\tiny_dict.txt | ForEach-Object {
    $bytes = [Text.Encoding]::UTF8.GetBytes($_)
    $md5   = [Security.Cryptography.MD5]::Create()
    ($md5.ComputeHash($bytes) | ForEach-Object { $_.ToString('x2') }) -join ''
} | Set-Content -Encoding ascii examples\sample_hashes.txt
```

### medium / big / brute scenarios

These are generated by a deterministic Rust binary that lives at [`crates/gpuhash-core/examples/gen_examples.rs`](../crates/gpuhash-core/examples/gen_examples.rs). Run from the repo root:

```powershell
cargo run --release --example gen_examples -p gpuhash-core
# or with a different seed:
cargo run --release --example gen_examples -p gpuhash-core -- 1234
```

The generator uses `gpuhash-core`'s own `digest()` for hashing, so any future algorithm fix automatically flows into the example files on the next regen. Output is byte-identical across runs with the same seed (default 42).
